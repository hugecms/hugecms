# HugeCMS 产品规格文档

> 状态：草案  
> 目标读者：开发者（同时也是项目负责人）  
> 核心诉求：用最低成本、最快速度把客户需求变成可交付功能，并保证长期可维护。

---

## 1. 项目定位

HugeCMS 是一个面向企业客户的**通用内容管理系统（CMS）**，基于 Laravel + Filament 构建。

### 1.1 目标用户

- **直接使用者**：企业客户的运营、编辑、管理员。
- **购买者/决策者**：中小企业主、市场负责人、行政/信息化部门。
- **系统交付者**：本项目开发者，需要快速响应客户定制需求。

### 1.2 核心卖点

1. **交付快**：常见 CMS 需求通过 Filament 配置即可实现，少写重复代码。
2. **易扩展**：内容模型、自定义字段、菜单权限可配置，覆盖多行业场景。
3. **维护成本低**：基于 Laravel 成熟生态，文档、人才、第三方包丰富。
4. **私有化部署**：可交付到客户自有服务器，符合国内企业采购习惯。

## 2. 技术栈

| 层级 | 技术 | 说明 |
|---|---|---|
| 后端框架 | Laravel 13.20 | 主框架，PHP ^8.3 |
| 管理后台 | Filament 5.7 | 基于 Livewire 的后台 UI，CRUD 开箱即用 |
| 数据库 | MySQL / PostgreSQL | 默认 MySQL，测试用 SQLite |
| 前端样式 | Tailwind CSS v4 | 通过 Vite 插件引入 |
| 构建工具 | Vite 3 + laravel-vite-plugin | 资源打包 |
| 媒体库 | Spatie Laravel Media Library | 已集成，支持多模型封面 |
| 代码风格 | Laravel Pint | 默认预设 |
| 测试 | PHPUnit | SQLite in-memory |

**决策原则**：不追新、不拆分微服务、不搞纯 Headless。优先保证单人开发和快速交付。

## 3. 核心功能模块

后台菜单分为 **6 个一级导航**，覆盖 80% 以上企业官网/内容站需求。

### 3.1 仪表盘

- 数据概览：文章总数、用户总数、今日访问量、待审核内容数。
- 快捷操作：新增文章、新增页面、查看待办。
- 最近动态：最近登录用户、最近发布内容、最近操作日志。

### 3.2 内容管理

CMS 的核心模块，必须支持灵活的内容模型。

#### 3.2.1 文章管理

- 文章列表（标题、分类、作者、状态、发布时间、操作）。
- 文章新增/编辑：标题、摘要、正文（富文本）、封面图、分类、标签、作者、SEO 信息。
- 文章状态：草稿、已发布、已下线、待审核。
- 文章操作：发布、下线、置顶、推荐、删除、批量删除。
- 文章回收站：软删除恢复机制。

#### 3.2.2 分类管理

- 树形分类结构（支持多级）。
- 分类字段：名称、别名、父分类、描述、封面图、排序、SEO 信息。
- 分类与文章关联，支持一个文章属于多个分类（可选）。

#### 3.2.3 标签管理

- 标签列表、新增、编辑、删除。
- 标签字段：名称、别名、描述。
- 标签与文章多对多关联。

#### 3.2.4 页面管理

- 管理独立页面，如“关于我们”“联系我们”“服务条款”。
- 页面字段：标题、别名（URL slug）、内容、模板、SEO 信息、发布状态。
- 支持自定义模板选择。

#### 3.2.5 评论/留言

- 评论列表、审核、回复、删除。
- 评论状态：待审核、已通过、已拒绝、已删除。
- 支持按文章、按用户筛选。

#### 3.2.6 回收站

- 统一展示已软删除的文章、页面。
- 支持恢复和彻底删除。

### 3.3 用户与权限

#### 3.3.1 用户管理

- 用户列表、新增、编辑、禁用、删除。
- 用户字段：用户名、邮箱、手机号、密码、头像、角色、状态、注册时间。
- 支持按角色、状态筛选。

#### 3.3.2 角色管理

- 预置角色：超级管理员、编辑、运营、客服、审计。
- 支持自定义角色。

#### 3.3.3 权限管理

- 基于 Filament Policy 的细粒度权限。
- 每个 Resource 默认包含：查看、新增、编辑、删除、导出。
- 权限按菜单/资源分组。

#### 3.3.4 操作日志

- 记录关键操作：登录、登出、新增、编辑、删除、发布。
- 日志字段：操作人、操作类型、对象类型、对象 ID、IP、时间、变更内容摘要。
- 支持按用户、时间、操作类型筛选。

### 3.4 媒体资源

#### 3.4.1 图片库

- 图片上传、预览、删除。
- 支持拖拽上传、批量上传。
- 图片字段：文件名、尺寸、大小、上传时间、引用位置。

#### 3.4.2 文件库

- 管理 PDF、Word、Excel 等非图片文件。
- 支持文件下载链接生成。

#### 3.4.3 资源分类

- 为媒体资源建立分类，便于检索。

### 3.5 运营工具

#### 3.5.1 轮播图/广告位

- 管理首页轮播图、Banner 位。
- 字段：标题、图片、链接、排序、状态、生效时间。

#### 3.5.2 公告/通知

- 系统公告、弹窗通知管理。
- 字段：标题、内容、类型、生效时间、过期时间、状态。

#### 3.5.3 SEO 管理

- 站点 TDK（Title、Description、Keywords）配置。
- 文章/页面独立 SEO 设置。
- 友情链接管理。
- Sitemap 自动生成。

### 3.6 数据报表

#### 3.6.1 访问统计

- 总访问量、独立访客、PV/UV 趋势。
- 热门文章排行。
- 来源分析（如接入第三方统计可预留字段）。

#### 3.6.2 用户行为

- 新增用户趋势。
- 活跃用户统计。

#### 3.6.3 自定义报表

- 预留扩展位，后续可按客户需求增加。

### 3.7 系统设置

#### 3.7.1 站点信息

- 站点名称、Logo、ICP 备案号、版权信息、联系信息。

#### 3.7.2 菜单管理

- 后台菜单配置（预留扩展位）。
- 前台导航菜单配置。

#### 3.7.3 存储配置

- 本地存储、OSS、S3 等配置项预留。

#### 3.7.4 邮件/短信配置

- SMTP、短信服务商配置项预留。

#### 3.7.5 第三方登录

- 微信、企业微信、钉钉等 OAuth 登录配置项预留。

## 5. 非功能需求

### 5.1 性能

- 后台列表默认分页 25 条，支持 50/100 条切换。
- 前台查询使用 Eager Loading，避免 N+1。
- 图片支持缩略图生成，减少前台加载时间。

### 5.2 安全

- 后台强制使用 HTTPS（生产环境）。
- 密码使用 bcrypt 哈希存储。
- 表单请求必须验证，防止 SQL 注入和 XSS。
- 敏感操作记录操作日志。

### 5.3 可维护性

- 使用 Laravel Pint 保持代码风格一致。
- 每个 Resource/Model 配套 Feature 测试。
- 配置项集中管理，避免硬编码。

### 5.4 可扩展性

- 内容模型预留自定义字段扩展机制。
- 菜单和权限按角色隔离，便于多租户升级。
- 第三方服务接入通过配置驱动，便于替换。

## 6. 交付节奏（MVP 建议）

### 第一阶段：核心后台（1~2 周）

- 仪表盘
- 文章管理、分类管理、标签管理
- 页面管理
- 用户管理、角色管理、权限管理
- 媒体库
- 站点设置

### 第二阶段：运营增强（第 3~4 周）

- 轮播图/广告位
- 公告/通知
- SEO 管理
- 友情链接
- 评论/留言
- 操作日志

### 第三阶段：数据与扩展（第 5~6 周）

- 访问统计
- 用户行为报表
- 自定义字段机制
- 多站点/多租户能力（可选）

---

## 7. 关键决策

1. **不使用纯 Headless 架构**：目标客户需要私有化部署和快速定制，传统单体更适合。
2. **前台初期用 Blade/Livewire**：减少前后端割裂，开发者单人可维护。
3. **权限模型采用 RBAC**：角色 + 权限策略，覆盖常见企业客户组织架构。
4. **自定义字段作为第二阶段重点**：这是区分通用 CMS 和行业垂直方案的关键能力。

## 8. 成功标准

- [ ] 单人可在 2 周内交付一个可用的企业官网 CMS。
- [ ] 常见客户需求（栏目调整、字段增减、权限分配）可在 1 天内响应。
- [ ] 系统运行稳定，单表百万级数据查询不卡顿。
- [ ] 新客户部署成本低于 2 小时。
